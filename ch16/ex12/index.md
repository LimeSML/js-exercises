## C10K問題とは
`Apache HTTP Server` などの Web サーバソフトウェアとクライアントの通信において、クライアントが約1万台に達すると、Webサーバーのハードウェア性能に余裕があるにもかかわらず、レスポンス性能が大きく下がる問題のこと

### 原因
`Apache` はクライアント接続ごとにプロセスを生成する方式のため、大量アクセス時にプロセス数の上限（約 3 万）に達し、新しい接続を処理できなくなる。これが C10K 問題の主因。

### 回避方法
- `Nginx` など、C10K 問題が起こらないように設計されたソフトウェアを使用する
- サーバーサイドで `Node.js` などの駆動方式を持ったソフトウェアを使用する
